## Wired in this pass

All wirings are defensive (nil-checks, panic recovery, no error path that can
kill the loop) and additive — visible TUI behavior is unchanged unless the
plan calls for it.

| Package / type | Call site | What now happens |
|---|---|---|
| `agent.Forethinker.Assess` | `internal/agent/react.go:104`, `internal/agent/stream.go:187` (via `agent.emitImpact` in `agent.go`) | Before every tool call, the agent computes an `ImpactAssessment` and publishes a `tool_impact` event with risk level, protected-path flag, affected paths, and reasoning. Plugins, journal, and the bus all see it. |
| `agent.AssessConfidence` | `internal/agent/agent.go:Run` final-step branch and `internal/agent/stream.go` Done branch (via `agent.assessTurnConfidence`) | Every successful turn attaches a `*ConfidenceAssessment` to `RunResult.Confidence`. |
| `agent.BudgetEstimator` | `internal/agent/agent.go:checkBudget` invoked at top of each `Run`/`Stream` step | When `ShouldWarn` flips true, agent emits `budget_warning` event with utilization, tokens, and `should_compact` hint. |
| `agent.ErrorRecovery` | `internal/agent/agent.go:emitRecovery` invoked from `Run`'s step-error path and `Stream`'s provider-error path | On any step failure: builds a `RecoveryReport`, emits `recovery` event, persists lesson via `JournalEntryWriter` if one is set, then emits `error` event. |
| `agent.EventBus` | `internal/agent/agent.go:emit` now forwards every event to the internal bus in addition to the host callback. New `Agent.Bus()` accessor lets external subscribers (journal, plugins, future TUI surfaces) listen without going through the single eventFn slot. |
| `journal.FlightRecorder` | `cmd/ethos/tui.go` `buildTUIApp` after agent construction | Per-session flight recorder lives at `~/.ethos/journal/raw/YYYY-MM-DD.jsonl`. `journalEventAdapter.Handle` forwards `tool_call`, `error`, `tool_impact`, `budget_warning`, `compact`, `recovery` events into the journal. The same adapter implements `agent.JournalEntryWriter` for recovery lessons via `agent.SetRecoveryWriter`. |

## Discoveries outside audit scope

1. **`app.Journal` was a field with no setter.** `pkg/tui/app.go` declared
   `Journal *journal.FlightRecorder` but no caller in the codebase ever set
   it. This is now wired in `cmd/ethos/tui.go`.
2. **`agent.SetEventFn` is single-slot.** Only one consumer can register at a
   time. The new internal `EventBus` provides multi-subscriber fanout; plugin
   manager + journal can both subscribe without stomping on each other once
   the plugin manager is migrated to use `Agent.Bus()` instead of the legacy
   eventFn (left for a follow-up since plugin code wasn't in scope here).
3. **Skills are loaded into `app.Skills` but never handed to the agent.** The
   loader populates the slice, but neither the system prompt nor any tool
   knows about enabled skills. Wiring this requires both a system-prompt
   augmentor and a skill-invocation tool; deferred until the skill picker
   dialog redesign lands.
4. **`internal/security/permission.go` (escalation) appears wired** through
   the existing approval cascade in `agent.approvalCheck`; no change needed.
   `internal/security/scanner.go` is wired in `cmd/ethos/tui.go:248`.
